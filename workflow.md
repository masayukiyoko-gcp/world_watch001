# プロジェクト開発ワークフロー（例：世界時計プロジェクト）

## 1. フォルダ構成・初期セットアップ
1.1 一般的なソフトウェアプロジェクトのフォルダ構成を作成する。
　- src/（ソースコード）
　- tests/（テストコード）
　- docs/（ドキュメント）
　- scripts/（スクリプト）
　- .github/（GitHub用設定）
1.2 必要に応じてサブフォルダ（modules, utils, config, unit, integration, design等）も作成する。

## 2. 仕様・要件定義
2.1 chater.md（プロジェクトの目的や背景、初期メモ）を作成する。
2.2 spec.md（仕様書案）を作成し、目的・機能・非機能要件・テスト方針を明確化する。
2.3 不明点や追加要件はCopilotに質問し、spec.mdへ随時反映する。

## 3. 開発環境構築
3.1 ローカルでWebアプリの動作確認ができるよう、簡易サーバ（Python http.serverやNode.js serve等）を用意する。
3.2 scripts/start_local_server.mdに起動手順を記載する。

## 4. 公開・運用方針
4.1 静的サイトとして構築し、公開用サーバはGitHub Pagesを利用する。
4.2 動的サイト構築は次回以降のプロジェクトで検討する。

## 5. Copilotとのやり取り
5.1 仕様や要件の明確化、設計・実装・テスト・運用の各フェーズでCopilotに相談し、内容をspec.mdやworkflow.mdに反映する。
5.2 無駄なやりとりや重複は省き、ワークフローの有効性を高める。
