# 仕様書案（spec.md）

## 1. 概要
本ドキュメントは、本プロジェクトのソフトウェア仕様をまとめたものです。

### 1.1 プロジェクトの目的
私が、copilotを活用したソフトウェア開発手法を習得すること。

## 2. システム構成
- src/ : ソースコード
  - modules/ : 主要機能モジュール
  - utils/ : ユーティリティ
  - config/ : 設定ファイル
- tests/ : テストコード
  - integration/ : 結合テスト
### 1.2 本プロジェクトの方針
- 本プロジェクトでは「静的サイト」として世界時計Webアプリを構築する。
- 公開用WebサーバはGitHub Pagesを利用する。
- 動的サイト構築は次回以降のプロジェクトで検討する。
- docs/ : ドキュメント
  - design/ : 設計資料
- scripts/ : スクリプト
- .github/ : GitHub用設定

## 3. 機能要件
- 世界の都市名を指定することで、現地の時刻をアナログ時計で表示する。
- 時計の数は動的に2~8まで変更可能。
- 時計の文字盤の色で、昼間か夜間か分かる機能を有する。
- 都市名の指定は、部分一致による候補をリストして選択できる機能を有する。
- 表示言語（日本語、英語、タイ語、インドネシア語）をプルダウンで選択・変更できる機能を有する。

## 3.1 UIレイアウト要件
・世界地図を背景に主要都市名を表示し、下部に各都市のアナログ時計を並べて表示する。
・時計の下には都市名、午前/午後表示、時計の数を増減するボタン、言語選択用プルダウン（日本語・英語・タイ語・インドネシア語）を配置する。
・都市の選択や時計の追加・削除、言語切替が直感的に行えるインターフェースとする。

## 4. 非機能要件
- copilotによるテストの容易性
- webアプリとwindowsローカルアプリの両方を作成するので、ソフトウェア部品の可用性

## 5. テスト方針
- 単体テストの条件カバレッジC0、C1
- 表示系の正しさの確認
- Webアプリの動作確認のため、ローカルサーバ（外部公開しない）を構築し、テストを実施する。
  - Pythonのhttp.serverやNode.jsのserve等を利用し、srcディレクトリ配下をルートとして起動する。
  - 詳細はscripts/start_local_server.mdを参照。

## 6. その他
- 特にありません。

## 7. PR ルール（必須）
- **ロジックの変更**: プルリクエストには、ロジック（時刻計算など）に対する単体テストを含めること。テストは条件網羅性（C0）および分岐網羅性（C1）で100%を達成していることをCIで検証します。
- **表示系の変更**: 表示に関する変更（UI/見た目）を含むPRは、Cypress等の自動テストに加え、意図した表示がされていることを示すスクリーンショットをPRに添付してください。CIはスクリーンショットの存在を確認します。
- **実行者**: 各Issueの実行主体は`copilot`（自動化アクタ）とし、Issueにラベル`executor: copilot`を付与します。GitHub上でのAssigneeが存在しない場合はこのラベルで実行者を示します。
- **期日**: すべての現行Issueの期日は `2026-03-15 24:00 (JST)` とし、マイルストーンを設定して管理します。
