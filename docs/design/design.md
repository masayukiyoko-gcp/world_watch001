# 設計資料（design.md）

## 1. システム概要
本プロジェクトは、世界の都市名を指定して現地時刻をアナログ時計で表示するWebアプリを開発します。静的サイトとしてGitHub Pagesで公開し、ローカル開発・テスト環境も整備します。

## 1.1 UIレイアウト案
添付イメージのように、世界地図を背景に主要都市名を表示し、下部に各都市のアナログ時計を並べて表示します。
時計の下には都市名、午前/午後表示、時計の数を増減するボタン、言語選択用プルダウン（日本語・英語・タイ語・インドネシア語）を配置します。
都市の選択や時計の追加・削除、言語切替が直感的に行えるインターフェースとします。

## 2. システム構成
- src/
  - modules/ : 主要機能モジュール
  - utils/ : ユーティリティ
  - config/ : 設定ファイル
- tests/
  - unit/ : 単体テスト
  - integration/ : 結合テスト
- docs/
  - design/ : 設計資料
- scripts/ : スクリプト
- .github/ : GitHub用設定

## 3. 主要機能
- 都市名を指定し、現地時刻をアナログ時計で表示
- 時計の数は2～8個まで動的に変更可能
- 昼夜で文字盤の色を変化
- 都市名の部分一致による候補リスト表示・選択
- 表示言語（日本語、英語、タイ語、インドネシア語）をプルダウンで選択・変更可能
- テスト容易性・ソフトウェア部品の再利用性

## 4. 開発環境
- Python 3.x（ローカルサーバ・スクリプト・自動テスト）
- 仮想環境（venv, conda等推奨）
- GitHubリポジトリ管理、CI/CD、GitHub Pages公開
- VS Code（推奨拡張：Python, GitHub Pull Requests, Prettier, ESLint等）
- ターミナル：bash, PowerShell
- ローカルサーバ：Python http.server, Node.js serve等
- Docker等による仮想化も検討可

## 5. テスト方針
- 単体テスト：C0, C1カバレッジ
- 表示系の正しさ確認
- ローカルサーバでの動作確認
- 詳細はscripts/start_local_server.md参照

## 6. インフラ・自動化
- DockerfileやGitHub Actionsで自動化対応
- 必要に応じてIaC化
